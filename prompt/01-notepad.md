이 프로젝트는 윈도우OS 데스크탑 컨셉을 경량화된 웹OS UX 어플리케이션으로 구현한것으로 
사용자는 브라우저에서 데스크탑 환경을 경험할 수 있는 목표로 제작되고 있으며

기존 프로젝트를 분석을 먼저진행후,  추가기능을 구현해주세요

## 추가기능
- 메모장 어플리케이션을 추가하려고 합니다.
  - 메모장 어플리케이션은 텍스트 파일을 생성, 열기, 편집, 저장할 수 있어야 합니다.
  - 메모장 어플리케이션은 창으로 열려야 합니다.
  - 메모장 어플리케이션은 기본적인 텍스트 편집 기능(복사, 붙여넣기, 잘라내기 등)을 제공해야 합니다.
  - 메모장 어플리케이션은 다중 문서 인터페이스(MDI)를 지원하여 여러 개의 메모장을 동시에 열 수 있어야 합니다.
  - 메모장 어플리케이션은 사용자가 작성한 내용을 자동으로 저장하는 기능을 포함해야 합니다.
  - 메모장 어플리케이션은 사용자 인터페이스가 직관적이고 사용하기 쉬워야 합니다.
  - 메모장 어플리케이션은 브라우저의 로컬 스토리지를 사용하여 데이터를 저장해야 합니다.
  - 메모장 어플리케이션은 반응형 디자인을 적용하여 다양한 화면 크기에서 잘 작동해야 합니다.
  - 메모장은 기존 text를 포함, md style을 추가로 지원합니다.
- 스토리지에 대한 설명
  - 사용자는 첫 접속시 고유한 세션ID가 부여되며 이 세션ID를 기반으로 로컬스토리지에 데이터를 저장합니다.
  - 로컬스토리지는 브라우저의 localStorage API를 사용합니다.
  - 웹의 루트 스토리지는 c:/webroot/ 으로 가정하며 세션ID가 abc123인 사용자가 "notes.txt"라는 파일을 저장하면 
    - 로컬스토리지 키: "notes.txt"
    - 로컬스토리지 값: 파일의 내용
    - c:/webroot/abc123/notes.txt 형태로 표현됩니다.
      - 폴더 생성시 : "abc" 생성시
      - c:/webroot/abc123/abc/notes.txt 형태로 표현됩니다.
- 내컴퓨터
  - 내컴퓨터 아이콘을 더블클릭시 파일 탐색기 창이 열리도록 구현해주세요.
  - txt파일은 메모장 어플리케이션으로 열리도록 구현해주세요.
  - 이후 다양한 확장자가 추가되어 어플리케이션에 연결될수 있음으로 확장자와 어플리케이션 매핑구조를 설계해주세요.

# 로컬 테스트
- 구현이 다되며 docker-compose.yml , deskweb-dev 모드로 실행해주세요
  - 실행및 빌드시 발생하는 오류를 모두 해결해주세요
  - 실행이되면 브라우저에서 http://localhost:8080 접속후 기능이 정상동작하는지 확인해주세요
  - 실행 확인을 위해 개발모드에서 작동이 잘되는지 확인을 위한 로그를 개발시 꼼꼼히 작성해주세요
  - 코드수정시 도커재시작없이 docker exec deskweb-dev qx compile 를 활용해주세요 - dev모드 
- 기본 로컬테스트가 수행되면 직접확인후 피드백 예정으로 추가 수정이 있을수 있습니다. 작동중인 어플리케이션을 유지해주세요
