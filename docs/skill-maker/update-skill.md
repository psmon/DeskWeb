# DeskWeb 스킬 업데이트 가이드

## 스킬 위치
```
.claude/skills/
├── deskweb-convention/SKILL.md   # 프로젝트 컨벤션
├── deskweb-app/SKILL.md          # 일반 앱 패턴
├── deskweb-game/SKILL.md         # 게임 앱 패턴
└── deskweb-llm/SKILL.md          # LLM/AI 통합 패턴
```

## 업데이트 절차

### 1. 프롬프트 히스토리 분석
- `prompt/deskweb/00-{지침}.md` ~ `999-{지침}.md` 에 프로젝트 개발 프롬프트가 넘버링 순으로 존재
- 스킬 활용 이전에 작성된 프롬프트라 중복 지침이 많음
- 의미있는 새로운 패턴이 발견되면 해당 스킬에 반영

### 2. 업데이트 판단 기준

**새 스킬 추가가 필요한 경우:**
- 기존 4개 카테고리에 속하지 않는 완전히 새로운 패턴
- 예: 네트워크/멀티플레이어, PWA/오프라인, 플러그인 시스템 등

**기존 스킬 업데이트가 필요한 경우:**
- 새로운 앱 유형이 추가되어 참조 패턴 테이블에 추가 필요
- 빌드/배포 방식 변경
- 새로운 외부 라이브러리 통합 패턴
- 버그 해결 과정에서 발견된 주의사항

### 3. 업데이트 수행

```
1. 기존 스킬 읽기: .claude/skills/{skill-name}/SKILL.md
2. 프롬프트 히스토리 확인: prompt/deskweb/ 하위 최신 파일
3. 소스 코드 확인: deskweb/source/class/deskweb/ 실제 구현 참조
4. 스킬 파일 수정: 변경사항 반영
5. 문서 업데이트: docs/skill-maker/readme.md 의 베스트 프롬프트 예시 갱신
```

### 4. 검증
- 업데이트된 스킬의 description이 검색 키워드를 충분히 포함하는지 확인
- SKILL.md 본문이 5,000 단어 이하인지 확인 (토큰 효율성)
- 실제 프로젝트 소스와 스킬 내용이 일치하는지 확인

## 업데이트 이력 관리
- 스킬 YAML frontmatter의 `date` 필드를 업데이트 날짜로 갱신
- 주요 변경사항은 이 파일 하단에 기록
