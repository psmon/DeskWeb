## 베스트 프롬프트 예시

### 1. 일반 앱 추가

**PDF 뷰어 앱 추가:**
```
PDF 뷰어 앱을 추가해주세요.
- pdf.js 라이브러리를 활용
- 파일탐색기에서 .pdf 더블클릭시 열리도록 확장자 매핑
- 페이지 탐색, 확대/축소 지원
```

**계산기 앱 추가:**
```
Windows 스타일 계산기 앱을 추가해주세요.
- 기본/공학 모드 전환
- 키보드 입력 지원
```

### 2. 게임 앱 추가

**2D 보드게임:**
```
오목 게임을 추가해주세요.
- 15x15 격자, Canvas 기반 렌더링
- AI 대전 모드 (LLM API 활용)
- 난이도 3단계 (쉬움/보통/어려움)
```

**3D 게임:**
```
Three.js 기반 볼링 게임을 추가해주세요.
- 3D 핀/공 렌더링, 물리 엔진 적용
- 카메라 각도 조절
- 사운드 효과
```

### 3. LLM 기능 추가

**챗봇 기능 확장:**
```
AI ChatBot에 이미지 생성 기능을 추가해주세요.
- 사용자가 프롬프트 입력시 이미지 생성 API 호출
- 생성된 이미지를 대화창에 표시
- 마크다운 형식 응답 유지
```

**AI-OS 명령 추가:**
```
AI ChatBot에서 "메모장 열어서 오늘 일정 작성해줘" 명령을 처리하도록 개선해주세요.
- 앱 실행 + 텍스트 입력 자동화
- 실행 결과를 채팅창에 피드백
```

### 4. 프로젝트 유지보수

**기존 앱 버그 수정:**
```
지뢰찾기 게임에서 난이도 변경시 게임판이 갱신되지 않는 버그를 수정해주세요.
- 쉬움→어려움은 작동하지만 어려움→쉬움 전환시 문제 발생
```

**테마 커스텀:**
```
Windows 11 스타일 테마를 추가해주세요.
- 둥근 모서리, 투명 효과
- 테마 전환 옵션
```

## 프롬프트 작성 팁

1. **기존 참조 명시**: "솔리테어와 동일한 방식으로" 등 기존 앱 참조
2. **기능 범위 한정**: 한 번에 하나의 앱/기능에 집중
3. **외부 라이브러리 명시**: 활용할 라이브러리가 있으면 구체적으로 지정
4. **테스트 지침 포함**: 빌드/실행 확인 요청 포함
5. **버그 설명은 구체적으로**: 재현 경로, 예상 동작, 실제 동작 명시
